# Emacs Config Lesson 7: Custom Modeline — Your First Real Elisp Program

**Context for Claude**

This is the most programming-intensive lesson. They will write a fully custom modeline from scratch — not using doom-modeline or powerline, but their own. The modeline teaches: custom faces, format strings, conditionals, and how Emacs renders the mode line. By the end they will have written a ~40-line elisp program and will understand every line. This is where elisp starts to feel like programming, not just config.

Take this lesson slowly. Do not rush. Ask questions constantly. This is a real achievement.

**Goal:** Understand what the modeline is, write a custom mode-line-format, define custom faces, use format and propertize, write conditional display logic.

---

## What IS the modeline?

"At the bottom of each Emacs window is a line that shows information about the current buffer. It shows: the filename, whether it's modified, what major mode is active, and the line/column number."

"That entire line is controlled by a single variable: `mode-line-format`."

Ask: "Look at the bottom of your Emacs window. What information do you see there?"

"Everything you see is generated by code. And we're going to replace it with our own code."

---

## Understand mode-line-format first

```
C-h v mode-line-format
```

"Read the documentation. What type is this variable?" (a list — a complex list of strings, symbols, and forms that Emacs evaluates to produce the mode line)

"Here's the default value — it's complicated and messy:"

```elisp
("" mode-line-mule-info mode-line-client mode-line-modified
 mode-line-remote mode-line-frame-identification
 mode-line-buffer-identification ...)
```

"We're going to replace this entire thing with our own, simpler version."

## Plan our modeline before writing it

Ask: "What information do you want in your modeline?"

Guide toward a simple but complete set:
1. Whether the buffer is modified (★ or space)
2. The buffer name (filename)
3. The major mode (e.g., "Go" or "Emacs-Lisp")
4. Line : Column position
5. Maybe: git branch (we'll leave this as optional stretch)

"Let's design what it will look like:"

```
 ★ init.el  Emacs-Lisp  42:7 
```

or

```
   practice.txt  Text  1:0 
```

## Step 1: propertize — adding color to strings

"In the mode line, we can color individual pieces of text using `propertize`. It takes a string and attaches display properties to it."

```elisp
;; In *scratch*: evaluate this with C-x C-e to see it
(propertize "hello" 'face '(:foreground "#61AFEF" :weight bold))
```

"This makes 'hello' appear in blue and bold. The `face` property controls appearance."

Ask: "What is a 'face' in Emacs?" (a named or inline set of visual properties — foreground color, background, bold, italic, underline, etc.)

Ask: "What is `'face`?" (a quoted symbol — we're passing the symbol `face` as a property name, not evaluating it as a variable)

---

## Step 2: Define custom faces

"Instead of hard-coding colors inline, we define named faces. This way if someone uses a different theme, the faces can be overridden."

Have them add to init.el:

```elisp
;; ============================================================
;; CUSTOM MODELINE
;; ============================================================

;; Define faces for different parts of our modeline
(defface my/modeline-modified
  '((t :foreground "#E06C75" :weight bold))
  "Face for the modified indicator in the modeline.")

(defface my/modeline-buffer-name
  '((t :foreground "#61AFEF" :weight bold))
  "Face for the buffer name in the modeline.")

(defface my/modeline-mode
  '((t :foreground "#98C379" :weight normal))
  "Face for the major mode name in the modeline.")

(defface my/modeline-position
  '((t :foreground "#ABB2BF"))
  "Face for the line:column position in the modeline.")
```

Ask about `defface`:
- "What is `defface`?" (defines a new named face — C-h f defface to read more)
- "What is `'((t :foreground "#61AFEF" :weight bold))`?" (a list of face specifications. The `t` means 'for all display types'. The rest are properties: foreground color as hex, and weight bold.)
- "Why define named faces instead of inline colors?" (so themes can override them — a dark theme and light theme can both override my/modeline-buffer-name to appropriate colors)

## Step 3: Write helper functions for each piece

"Let's build each piece of the modeline as its own small function. This makes it easy to understand and modify later."

**Modified indicator:**
```elisp
(defun my/modeline-modified-indicator ()
  "Return ★ if buffer is modified, space if not."
  (if (buffer-modified-p)
      (propertize " ★ " 'face 'my/modeline-modified)
    (propertize "   " 'face 'my/modeline-position)))
```

Ask: "What does `buffer-modified-p` do?" (returns t if the current buffer has unsaved changes, nil if not. The `-p` suffix is a convention in elisp for 'predicate' — functions that return true or false.)

Ask: "What is `if` in elisp?" (like if/else in Go: `(if condition then-form else-form)`. If condition is non-nil, evaluates then-form. Otherwise evaluates else-form.)

Ask: "What does `propertize " ★ "` do vs just returning `" ★ "`?" (propertize attaches display properties — the color face — to the string. Without it, we'd get the ★ but with no special color.)

**Buffer name:**
```elisp
(defun my/modeline-buffer-name ()
  "Return the buffer name, styled."
  (propertize (buffer-name) 'face 'my/modeline-buffer-name))
```

Ask: "What does `buffer-name` return?" (the name of the current buffer as a string — for file buffers it's usually the filename)

**Major mode:**
```elisp
(defun my/modeline-mode-name ()
  "Return the major mode name, styled."
  (propertize (format-mode-line mode-name) 'face 'my/modeline-mode))
```

Ask: "What is `mode-name`?" (a variable set by each major mode that holds a human-readable name. `format-mode-line` evaluates it properly since mode-name can sometimes be a list.)

**Position:**
```elisp
(defun my/modeline-position ()
  "Return line:column position."
  (propertize
   (format " %d:%d " (line-number-at-pos) (current-column))
   'face 'my/modeline-position))
```

Ask: "What does `format` do in elisp?" (like `fmt.Sprintf` in Go — formats a string with placeholders. `%d` means integer, `%s` means string.)

Ask: "What does `line-number-at-pos` return?" (the current line number as an integer. `current-column` returns the column number.)

## Step 4: Assemble the modeline

```elisp
(defun my/modeline-render ()
  "Render the full custom modeline."
  (list
   (my/modeline-modified-indicator)
   " "
   (my/modeline-buffer-name)
   "  "
   (my/modeline-mode-name)
   "  "
   (my/modeline-position)))

;; Apply it globally
(setq-default mode-line-format '(:eval (my/modeline-render)))
```

Ask: "What does `list` do here?" (creates a list of the return values of all those functions — strings with properties)

Ask: "What is `'(:eval ...)` in the mode-line-format?" (a special form that tells Emacs to EVALUATE the expression every time it renders the mode line. Without :eval, Emacs would only call our function once at startup. With :eval, it calls it constantly — so the display updates as you move around.)

Ask: "Why is `mode-line-format` set with `setq-default`?" (it's buffer-local — each buffer has its own mode line format. setq-default applies to all new buffers. But we also want it to apply to existing buffers — guide them to notice this and fix it if needed with `(force-mode-line-update t)`)

## Teach what they built

"Stop. Look at this code. You just wrote:
- 4 custom face definitions (`defface`)
- 4 helper functions (`defun`)
- 1 render function that composes them
- 1 config line that wires it to Emacs

Every character in your modeline is generated by YOUR code. Every time you move the cursor, Emacs calls `my/modeline-render`, which calls those four helper functions, which return colored strings, which Emacs displays.

That's a real program. Not a config file — a program."

Ask: "How would you add the current time to the modeline?" 

Guide: use `(format-time-string "%H:%M")` — returns the current time as a string. They just need to add a new helper function and include it in my/modeline-render.

Ask: "How would you show the git branch?" (harder — requires checking if magit or built-in vc is available. Leave this as a stretch goal: `(when (fboundp 'magit-get-current-branch) ...)`)
