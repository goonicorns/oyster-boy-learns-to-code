# Lesson 31: abigen — Generate Go Code From a Contract

**For Claude — do not show this file to the learner**

---

## Context for Claude

In lesson 30 they called contracts manually — packing ABI data and unpacking raw bytes. It worked, but it's verbose and error-prone. `abigen` is a tool that reads an ABI JSON file and generates a proper Go package with typed methods. This is how real projects do it. This lesson: install abigen, generate bindings for ERC-20, then use those bindings to do everything from lesson 30 with a fraction of the code.

**This lesson's goal:**
- Understand why code generation exists
- Install and run `abigen`
- Read the generated code and understand what it did
- Call ERC-20 methods using the generated bindings
- Understand the tradeoff: generated code vs manual ABI calls

---

## Why code generation?

"In lesson 30, calling `totalSupply()` was maybe 15 lines. Pack, call, unpack, cast. Do that for every function on a complex contract — some have 50+ functions — and you've written thousands of lines of boilerplate. One typo in the function name or a wrong type crashes at runtime."

"Code generation solves this: you describe the contract once (the ABI), and a tool writes all the boilerplate Go code for you. Every function becomes a proper typed Go method. Compile-time safety."

Ask: "Can you think of another place in programming where code is generated from a schema?" — guide toward: Protobuf/gRPC generates Go code from `.proto` files. SQL migrations generate model code. This is the same idea.

## Run abigen

```bash
abigen --abi=contracts/erc20/erc20.abi --pkg=erc20 --out=contracts/erc20/erc20.go
```

Flags:
- `--abi` — the ABI file to read
- `--pkg` — the Go package name for the generated file
- `--out` — where to write the generated Go file

Now open `contracts/erc20/erc20.go`. Have them read it. It's long but structured. Ask questions as they scan:

- "Find the `Name` method. What does it return?" (a string and an error — no manual unpacking needed)
- "Find `BalanceOf`. What's its argument type?" (`common.Address` — not a string, a typed address)
- "Find the `Transfer` event struct near the top. What fields does it have?" (From, To, Value — all typed)
- "What is `bind.CallOpts`?" (options for the call — you can set a block number, a `From` address for simulation, etc.)

## Use the generated bindings

New file `contracts/erc20/reader.go`:

```go
package erc20

import (
    "context"
    "fmt"
    "math/big"

    "github.com/ethereum/go-ethereum/accounts/abi/bind"
    "github.com/ethereum/go-ethereum/common"
    "github.com/ethereum/go-ethereum/ethclient"
)

func PrintTokenInfo(client *ethclient.Client, contractAddr string) error {
    addr := common.HexToAddress(contractAddr)

    // NewERC20 is generated by abigen — it creates a bound contract instance
    token, err := NewERC20(addr, client)
    if err != nil {
        return fmt.Errorf("binding contract: %w", err)
    }

    opts := &bind.CallOpts{Context: context.Background()}

    name, err := token.Name(opts)
    if err != nil {
        return fmt.Errorf("name: %w", err)
    }

    symbol, err := token.Symbol(opts)
    if err != nil {
        return fmt.Errorf("symbol: %w", err)
    }

    decimals, err := token.Decimals(opts)
    if err != nil {
        return fmt.Errorf("decimals: %w", err)
    }

    totalSupply, err := token.TotalSupply(opts)
    if err != nil {
        return fmt.Errorf("totalSupply: %w", err)
    }

    divisor := new(big.Float).SetInt(
        new(big.Int).Exp(big.NewInt(10), big.NewInt(int64(decimals)), nil),
    )
    human := new(big.Float).Quo(new(big.Float).SetInt(totalSupply), divisor)

    fmt.Printf("%-12s %s\n", "Name:", name)
    fmt.Printf("%-12s %s\n", "Symbol:", symbol)
    fmt.Printf("%-12s %d\n", "Decimals:", decimals)
    fmt.Printf("%-12s %.2f %s\n", "Supply:", human, symbol)

    return nil
}

func PrintBalance(client *ethclient.Client, contractAddr, holderAddr string) error {
    token, err := NewERC20(common.HexToAddress(contractAddr), client)
    if err != nil {
        return fmt.Errorf("binding contract: %w", err)
    }

    opts := &bind.CallOpts{Context: context.Background()}
    balance, err := token.BalanceOf(opts, common.HexToAddress(holderAddr))
    if err != nil {
        return fmt.Errorf("balanceOf: %w", err)
    }

    decimals, _ := token.Decimals(opts)
    divisor := new(big.Float).SetInt(
        new(big.Int).Exp(big.NewInt(10), big.NewInt(int64(decimals)), nil),
    )
    human := new(big.Float).Quo(new(big.Float).SetInt(balance), divisor)

    symbol, _ := token.Symbol(opts)
    fmt.Printf("Balance: %.6f %s\n", human, symbol)
    return nil
}
```

Compare to lesson 30. Ask:
- "How many fewer lines is this version?" (roughly half)
- "Where did the ABI parsing go?" (into the generated file — abigen embedded it)
- "Where did Pack/Unpack go?" (also generated — `token.Name(opts)` does all of it internally)
- "What's `bind.CallOpts` vs `context.Background()`?" (CallOpts wraps the context plus optional block number and caller address)

## Run it against multiple tokens

In `main.go`, add calls against different ERC-20 tokens:

```go
const (
    USDC = "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48"
    USDT = "0xdAC17F958D2ee523a2206206994597C13D831ec7"
    LINK = "0x514910771AF9Ca656af840dff83E8264EcF986CA"
)
```

"Run PrintTokenInfo on all three. Notice anything different about LINK's decimals vs USDC?" (LINK uses 18 decimals like ETH; USDC uses 6)

Ask: "Why would a stablecoin use 6 decimals instead of 18?" (1 USDC ≈ $1, so sub-cent precision is enough; 18 decimals would waste storage for almost no practical benefit)
